Sean Behan's CS Career

Aspiring Software Systems Engineer

Ever since I was little, I was the kid who would buy a Lego set, follow the instructions once, then tear it apart and build something completely different. No blueprint, no defined path, just figuring it out as I went. I think that mindset is exactly why software engineering clicked for me the way it did.
I love to code. Whether it is building a full stack web app, designing a front end experience, or writing logic for a simple game, I genuinely enjoy sitting in the details and solving the problem in front of me. But what excites me even more than writing code is the system around it. The software development cycle, the tooling, the workflows that determine how fast and how well a team can ship software.
At my co-op at Bevi, I got a real taste of this. Using tools like CircleCI, Testmo, Slack, and AI to optimize testing pipelines showed me how much the right infrastructure impacts not just the engineering team, but the pace of an entire company. That intersection of code, systems thinking, and tooling is where I want to build my career.
I am also drawn to the user side of software. Finding ways to take raw data and turn it into something intuitive and meaningful for the person using it is a challenge I find deeply interesting. My goal is to work as a software systems engineer or front end engineer, with a focus on building better workflows and better experiences in the age of AI.

My Industry Experience

Role

I came in as a Software Engineer in Test but the role turned into something bigger than that pretty quickly. My core responsibility was owning the automated testing pipeline, but I ended up working across product, design, and hardware teams to validate functionality at every level. That cross-functional exposure gave me a much broader view of how software actually ships inside a real company.

Workflow Optimization

A big part of what I focused on was making the testing workflow faster and more reliable. Using CircleCI I built and maintained a CI/CD pipeline integrated with GitHub that automatically builds APKs, distributes them across Android emulators, and runs conditional test suites based on branch type. I also worked heavily with Xray, Testmo, and Jira to manage test cases, track coverage, and keep the team aligned on quality across sprints. The goal was always the same: reduce the time between a code change and a confident deployment.

Languages and Libraries

I worked across two of Bevi's front end projects, writing automated UI tests in both Kotlin with the Espresso testing framework and TypeScript with Playwright. Both required building foundational test architecture from scratch using OOP principles and the Page Object Model to keep tests maintainable as the product grew.

Agentic Coding with Claude

The work I am most proud of from this co-op was building an agentic testing workflow using Claude Code. I created Claude skills that take a feature description and automatically generate structured test cases, then pass those into a second skill that writes the actual automation code. I built this for both the Kotlin and TypeScript projects, so the same pipeline could produce Espresso tests or Playwright tests depending on the context. It dramatically cut the time from a new feature to full test coverage and is one of the clearest examples I have seen of AI making a real difference in a software development workflow.

What is EuroTour?

While abundant public data exists on road quality, GDP, tourism metrics, and regional factors, no single platform connects these insights meaningfully. What is Sanguine?

Sanguine is a two player card game where you place cards on a board to control cells and increase your points for the row. Based on Queen's Blood from Final Fantasy, the game logic is separated from the view making it easy to test and change. Two players take turns placing cards, each affecting nearby cells based on its influence pattern. Whoever has the highest card values in each row wins points from that row.

What is Connections?

Connections is a word puzzle game where you sort 16 words into four groups of four, each sharing a hidden common thread. The words are carefully chosen to mislead you — a word may look like it belongs in one group when it fits somewhere else entirely. You only get four mistakes before the game ends. The four groups are color coded by difficulty, from straightforward yellow to the notoriously tricky purple.

How is AI involved?

AI has genuinely changed how I work as a developer. At my co-op at Bevi, I use it to accelerate understanding of unfamiliar code, pressure-test my own approaches before implementing them, and compress the feedback loop on debugging. On the project side, building Rundown would have taken me twice as long without it. I came in knowing Java but React Native, Supabase Edge Functions, and OAuth flows were all new territory. AI let me learn by doing instead of learning by reading docs for hours before writing a single line. The key thing I've figured out is that it works best as a thinking partner, not an answer machine. When I explain my problem and reasoning first, I actually understand what comes back. That habit has made me a sharper engineer overall.
